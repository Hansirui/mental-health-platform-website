# 基于多模态感知与知识推理的抑郁症智能评估与干预系统

## 一、项目简介

本项目面向大学生心理健康场景，围绕抑郁倾向初筛、个性化建议生成与跟踪复评，构建一个融合多模态感知、知识图谱推理与隐私保护的智能评估与干预原型系统。

项目希望解决传统心理健康评估中存在的几个典型问题：一是单一问卷或单一信息源导致评估结果稳定性有限；二是心理健康数据敏感，存在隐私泄露风险；三是很多系统只能给出结果，难以提供具有可解释性的建议路径。

因此，本项目拟通过整合问卷、文本、语音等多源信息，并结合抑郁症领域知识图谱，形成“评估—建议—跟踪—复评”的闭环服务流程，提升心理健康服务的智能化、个性化与可扩展性。

---

## 二、研究目标

1. 构建融合问卷、文本、语音等多模态信息的抑郁倾向智能评估方法。
2. 建立“症状—诱因—干预—禁忌”四维知识图谱，提升系统建议生成的可解释性。
3. 探索差分隐私、边缘计算等技术在心理健康场景中的应用，降低敏感数据使用风险。
4. 形成可运行的原型系统，实现评估、推理、建议和跟踪复评的完整闭环。

---

## 三、研究内容

### 1. 多源异构数据治理与动态知识图谱构建
围绕 DSM-5、临床指南、医学文献、公开语料及量表数据等多类数据源，完成清洗、去重、术语统一与结构化整理，设计“症状—诱因—干预—禁忌”四维本体，构建抑郁症领域知识图谱，为后续推理与建议生成提供知识底座。

### 2. 面向隐私保护的多模态评估
结合 PHQ-9 问卷、文本输入与后续可扩展的语音特征，开展多模态评估研究；在系统设计中考虑差分隐私、边缘计算等方法，尽量实现敏感数据“可用不可见”。

### 3. 知识推理驱动的建议生成
将用户评估结果映射到知识图谱，通过图谱查询与规则约束生成“主要症状—可能诱因—建议措施—禁忌提醒”的可解释路径输出，提升系统建议的针对性与可信度。

### 4. 原型系统集成与验证
采用前后端分离架构完成原型系统开发，集成评估模块、知识图谱模块、结果展示模块、历史记录模块与趋势分析模块，并通过功能测试、性能测试、隐私安全测试与可用性测试持续优化系统。

---

## 四、项目技术路线

本项目整体遵循“数据与知识底座建设—隐私保护下的多模态评估—知识推理驱动的建议生成—系统化集成与验证”的研发链条。

具体流程如下：

1. 采集并整理规范知识源、公开文本样本、问卷数据及可扩展语音数据；
2. 构建抑郁症领域知识图谱，形成结构化知识底座；
3. 设计问卷评估、文本分析及后续多模态融合方案；
4. 将评估结果映射到知识图谱，生成个性化建议与解释路径；
5. 完成系统原型开发，实现评估、建议、跟踪、复评的闭环服务；
6. 通过系统测试与案例分析持续优化图谱、模型与交互体验。

## 五、系统设计思路

本项目采用前后端分离架构进行开发。

### 前端模块
前端主要负责用户交互与结果展示，当前已规划或实现的页面包括：

- 首页
- 问卷评估页
- 文本输入页
- 结果展示页
- 图谱展示页
- 历史记录页
- 趋势分析页
- 个人中心/隐私说明页

### 后端模块
后端主要负责业务逻辑处理与数据服务，后续将逐步实现以下接口能力：

- 用户管理
- 问卷提交与评分
- 文本评估
- 综合风险判断
- 知识图谱查询
- 个性化建议生成
- 历史记录存储与读取

### 数据与知识层
- 使用 Neo4j 存储知识图谱实体与关系网络；
- 使用 MySQL 存储用户信息、评估记录、图谱版本与系统业务数据；
- 通过后端统一接口实现图谱查询与建议生成。

## 九、项目特色

### 1. 多模态评估与知识推理结合
项目不仅关注问卷或单一模型的判断结果，还强调结合知识图谱生成更具解释性的建议路径。

### 2. 隐私保护导向
项目在设计阶段即考虑差分隐私、边缘计算等隐私增强技术，强调心理健康敏感数据的合规使用。

### 3. 闭环服务思路
项目目标不是单点功能实现，而是构建“评估—建议—跟踪—复评”的完整闭环原型系统，更贴近真实心理健康服务流程。

### 4. 可扩展的工程架构
当前系统采用模块化、前后端分离设计，便于后续接入新量表、新模态、新推理策略以及更完整的数据库与接口体系。
